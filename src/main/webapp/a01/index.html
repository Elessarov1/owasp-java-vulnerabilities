Broken Access Control — IDOR Demo

This page demonstrates how an authenticated user can access another user's document when the backend loads an object only by its ID and does not verify ownership.

Demo Scenario

Document Access
Alice owns document 100. Bob owns document 200. The vulnerable endpoint loads a document only by ID and does not check ownership.
Vulnerable flow
The backend trusts the requested document ID and returns the document without verifying that it belongs to the current user.
Fixed flow
The backend checks the current authenticated user and returns only documents that belong to that user.
Alice can request Bob's document by changing the ID
Authorization is missing at the object level
The fixed endpoint enforces ownership checks
Unauthorized document access returns 404
Select a user and document ID, then send the same request to the vulnerable and fixed endpoints. Try Alice with document 200 to see the IDOR behavior.
