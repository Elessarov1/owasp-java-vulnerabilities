Injection — SQL Injection with JDBC

This page demonstrates how SQL built from user-controlled text changes query semantics, and how a prepared statement keeps the same input as data.

Demo Scenario

Search by username
The vulnerable endpoint formats the username into SQL executed by a JDBC Statement. The fixed endpoint binds it through a PreparedStatement.
Vulnerable flow
Input becomes part of the SQL command. The payload closes the string literal and adds a condition that matches every user.
Fixed flow
SQL structure is constant. JDBC binds the username to a placeholder, so SQL syntax in the value is not evaluated.
String formatting mixes untrusted data with SQL
The injected predicate returns all user records
Database data is exposed outside the intended search
Prepared SQL uses a ? placeholder
The username is bound with setString
The injection payload is searched as a literal username
First search for alice. Then use the prefilled injection payload. The vulnerable endpoint returns every account, while the fixed endpoint returns an empty result because no username equals the payload.
