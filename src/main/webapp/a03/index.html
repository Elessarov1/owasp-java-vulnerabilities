OWASP A03

Software Supply Chain Failures — Unsafe Template Interpolation

This page demonstrates how an outdated third-party dependency combined with dangerous API usage can turn user-controlled input into evaluated template expressions.

Demo Scenario

Dependency and API
The vulnerable endpoint uses org.apache.commons:commons-text:1.9 and passes user input directly to StringSubstitutor.createInterpolator().replace(...).
Vulnerable flow
The user controls the whole template string. The backend evaluates it as a template, not as plain text.
Fixed flow
The user selects a predefined server-side template and provides only simple values for known placeholders.
User input is interpreted as a template expression
Runtime details can be exposed through supported lookups
The vulnerable code uses broad default interpolators
Templates are selected from a server-side catalog
Only known placeholders are substituted
The fixed code does not use createInterpolator()
The vulnerable example is intentionally insecure. The goal is to show that a supply chain issue is often dangerous because of how the dependency is used in application code.
