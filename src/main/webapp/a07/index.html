OWASP A07

Authentication Failures — Password Spraying

This page demonstrates how unlimited login attempts let an automated attacker test password candidates, and how temporary account and client throttling interrupts the attack.

Demo Scenario

Demo account: alice
The password list ends with the valid password Summer2026!. Both implementations use BCrypt; the security difference is login-attempt limiting.
Vulnerable flow
Every request reaches password verification. The fifth candidate succeeds because previous failures have no effect.
Fixed flow
After three failures, further attempts receive 429 Too Many Requests and a Retry-After header for 30 seconds.
No failed-attempt counter
No delay or temporary block
Automated guessing continues until success
Failures tracked by account and request source
Limits checked before BCrypt verification
Generic errors do not reveal whether a user exists
A production authentication system should combine throttling with MFA, breached-password checks, secure session management, logging, alerting, and a trusted authentication framework.
