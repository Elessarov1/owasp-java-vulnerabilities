# Documentation

This directory contains general notes for secure code review, project structure, and video preparation.

## Files

- `project-philosophy.md` — explains the purpose and learning approach of this project.
- `code-review-checklist.md` — general checklist for reviewing Java web application code.

## Goal

The goal of these documents is to support the practical examples in the source code.

Each vulnerability example should include:

- vulnerable code;
- fixed code;
- HTTP requests;
- explanation;
- mitigation notes;
